Spring Boot

Install project

The following command installs Spring Boot project and skips the tests with --DskipTests flag. If tests are set up then just run the command without the flag. Run this command is necessary each time pom.xml is updated with new dependencies and/or aplication.properties is edited.

./mvnw clean install -DskipTests

The command is to be run in the main directory of the project, where mvnw and mvnw.cmd lay.

Keep in mind that if spring.jpa.hibernate.ddl-auto=update line inside aplication.properties is set up to create instead of update, the entire database is deleted and created again.

Run project

./mnvw spring-boot:run

Docker

Start Docker

Run the following command to start Docker.

sudo systemctl start docker

Dockerfile, building and running docker images

Once created the Dockerfile in your project, you must build it in order for containers and images to be created. You'll navigate to your project directory, where the Dockerfile is located, and run the following command in your terminal.

sudo docker build .

After successfully built, you'll need to check for docker images in order for you to be able to run the specific image you just created. You can check docker images by running

sudo docker images

Once you see the image Id of the image you're trying to run, execute the following command. Replace '8000' with the port as specified in your Dockerfile, namely, the port in which your application will be up and running. For instance, if you've dockerized and Angular application, usually the port will be 4200:4200 or 4200:4201. The next string is an example of an image ID. Replace it with the actual image ID you're trying to run.

sudo docker run -p 8000:8000 515ccc06601d

Specifying the port when running an image is IMPORTANT. Otherwise you won't be able to see it running and up.

Containers, removing images and containers

In case something went wrong when building an image or you have modified the Dockerfile heavily so you'll need to build it again, you'll need first to erase the containers and images associated. First, you'll have to check for all containers. You can do this by typing the following command.

sudo docker ps -a

In case your container is running, before deleting it you'll need to stop it. Run the following command to this purpose.

sudo docker stop the-container-id-you're-trying-to-delete

Thereafter, you can execute the following command to remove the container.

sudo docker rm container-id-1 container-id-2 ...

You can delete one container or a batch, like in the example above.

Finally, execute the following command to delete the images. You can erase several images at once, you'll just need the image Id. Type the following.

sudo docker rmi image-id-1 image-id-2 ...
